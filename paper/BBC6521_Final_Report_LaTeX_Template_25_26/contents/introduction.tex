\subsection{Background and Motivation}

The fifth-generation (5G) mobile network has shifted from the homogeneous ``one network fits all'' paradigm of previous generations towards a service-oriented architecture in which a shared physical infrastructure is partitioned into multiple logical networks through the use of network slicing~\cite{afolabi2018slicing}. Each slice is designed to satisfy the differentiated Quality of Service (QoS) requirements of a specific service class. The three canonical slice types defined by 3GPP -- Enhanced Mobile Broadband (eMBB), Ultra-Reliable Low-Latency Communication (URLLC), and massive Machine-Type Communication (mMTC) -- exhibit fundamentally different performance requirements: eMBB prioritises sustained high throughput, URLLC targets millisecond-level end-to-end latency together with stringent reliability guarantees, and mMTC is designed to accommodate the simultaneous access of a very large number of low-rate devices.

When these slice types coexist on a common pool of radio resources, their objectives become inherently competitive. The bandwidth consumed by a population of eMBB users streaming high-definition video may directly erode the latency budget available to URLLC flows, while bursty mass reporting by mMTC devices can exhaust access resources that other slices rely upon~\cite{afolabi2018slicing}. Traditional approaches to resource management -- rule-based heuristics and Deep Reinforcement Learning (DRL)-based optimisation -- have been widely studied in this setting~\cite{li2018drl,sciancalepore2019rlnsb}. Although these methods can encode hard constraints and priorities effectively, they struggle to integrate Service Level Agreement (SLA) semantics, operator policies and dynamically evolving user behaviour into a single adaptive decision-making framework.

The emergence of Large Language Models (LLMs) and the broader family of LLM-based autonomous agents has introduced an alternative line of investigation~\cite{zhou2024llmtelecom,wang2023agentsurvey}. Recent work has begun to examine the use of LLMs as high-level decision engines for wireless network management, motivated by their capacity for in-context reasoning, structured tool use and natural-language policy expression~\cite{tong2025wirelessagent,shao2024wirelessllm}. However, the application of LLM-based agents to network slicing remains at an early stage, and a number of architectural questions have not yet been addressed in a satisfactory manner.

\subsection{Problem Statement}

Two gaps in the current literature motivate the present study.

\textbf{Gap~1: Absence of an end-to-end multi-agent LLM framework for network slice resource management.} The WirelessAgent framework~\cite{tong2025wirelessagent} has demonstrated that an LLM endowed with a perception--memory--planning--action cognitive architecture can approach rule-optimal performance in slicing scenarios without any task-specific training. Nevertheless, WirelessAgent remains, at its core, a centralised single-agent design: one LLM is responsible for resource decisions across all slices, with no explicit specialisation of analytical roles or structured mechanism for mediating between the conflicting objectives of different slices. CommLLM~\cite{jiang2024commllm} introduces a three-component multi-agent architecture, but its validation is primarily targeted at semantic communication tasks rather than slice resource allocation. A complete framework dedicated to network slice resource management, supporting explicit multi-agent collaboration and validated end-to-end in a reproducible slicing environment, therefore remains underexplored.

\textbf{Gap~2: Lack of structured negotiation and conflict-resolution mechanisms.} Network slice resource allocation can be cast as a constrained multi-party optimisation problem in which total radio resources are fixed and the slices behave as competing stakeholders~\cite{caballero2019games}. This setting differs substantially from the predominantly cooperative task structures addressed by general-purpose multi-agent LLM frameworks such as MetaGPT~\cite{hong2023metagpt} or ChatDev~\cite{qian2023chatdev}. In competitive slicing scenarios, agents must reconcile conflicting objectives under hard resource constraints and service-level requirements, yet systematic negotiation protocols tailored to such environments are rarely discussed in the existing network management literature.

\subsection{Research Objectives}

The overarching objective of this project is to design, implement and evaluate an LLM multi-agent negotiation-based resource allocation framework for 5G network slicing, in which a low-frequency LLM multi-agent policy layer is coupled with a high-frequency rule-based execution layer, and to compare this framework with rule-based and DRL baselines in a reproducible simulation environment. This high-level objective is refined into two research questions.

\textbf{RQ1:} Can a multi-agent LLM framework outperform traditional rule-based strategies and DRL methods in network slice resource management? The associated objective is to quantify the performance differences between the proposed framework and representative baselines in terms of SLA satisfaction rate, resource utilisation and dynamic load adaptability.

\textbf{RQ2:} Does the negotiation mechanism between agents yield a structural advantage, as opposed to simply providing additional LLM computation? The associated objective is to compare a configuration that includes the full proposal--evaluation--negotiation protocol against an otherwise identical configuration in which the coordinator imposes allocations directly, in order to isolate the contribution of negotiation itself.

\subsection{Scope and Assumptions}

The scope of this work is deliberately constrained in order to ensure that the research questions can be addressed rigorously within the time available. The study is confined to a simulation-based evaluation using a custom Gymnasium~\cite{brockman2016gym} environment; no deployment on physical radio hardware or operational testbed is attempted. Three slice types -- eMBB, URLLC and mMTC -- are considered, following the canonical 3GPP classification. The LLM backbone is provided by a single commercial API (GPT-4) and the temperature is fixed to zero for all experiments in order to constrain reproducibility to environmental and DRL stochasticity. SLA definitions adopted in the simulation represent relative priority isolation rather than strict physical-layer 3GPP metrics; the implications of this abstraction are discussed explicitly in Chapter~3. Finally, the workload patterns used in the experiments are synthetic, reflecting the scarcity of publicly available slicing workload datasets with explicit SLA annotations.

\subsection{Contributions}

The contributions of this project are twofold.

\begin{enumerate}
    \item \textbf{A reproducible 5G network slicing simulation environment.} A custom Gymnasium-based environment is implemented, with clearly defined state and action spaces, a parameterised reward function and per-slice SLA evaluation logic. All parameters and interfaces are documented to support reproduction and extension.
    \item \textbf{A two-tier LLM multi-agent decision framework with an explicit negotiation protocol.} The upper tier consists of specialised slice-manager agents and a global coordinator agent that collaborate on resource allocation decisions through a three-phase proposal--evaluation--negotiation protocol; the lower tier executes rule-based cached strategies at millisecond granularity. The framework is compared against rule-based, DRL and single-agent LLM baselines through a set of controlled experiments designed to address RQ1 and RQ2.
\end{enumerate}

\subsection{Report Organisation}

The remainder of this report is organised as follows. Chapter~2 reviews the technical background and related work on network slicing architectures, SLA-aware scheduling, LLM-based agent frameworks for network management, and multi-agent coordination and negotiation mechanisms, and identifies the specific research gaps addressed in this work. Chapter~3 presents the design and implementation of the proposed framework, including the two-tier scheduling architecture, the internal logic of each agent role, the negotiation protocol, the Gymnasium-based simulation environment, the LangGraph-orchestrated workflow, and the baseline methods used for comparison. Chapter~4 describes the experimental design and reports the results obtained on the core performance comparison and the negotiation mechanism validation, together with a discussion of threats to validity and practical implications. Chapter~5 concludes the report with a reflection on the outcomes and an outline of possible directions for further work.
